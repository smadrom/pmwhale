# Security policy

## Supported versions

Until a stable release exists, only the latest commit on `main` and the latest
`0.1.x` release receive security fixes.

## Reporting a vulnerability

Please do not open a public issue. Use GitHub's private vulnerability reporting
for this repository. If that option is unavailable, contact the maintainer
at `sn.one.dev@gmail.com`.

Include the affected version or commit, impact, reproduction steps, and any
suggested mitigation. Do not include real credentials, private datasets, or
unredacted personal data. You should receive an acknowledgement within seven
days. Timelines for a fix and disclosure will depend on severity and complexity.

The project reads only public APIs and does not need secrets. A request to add
exchange credentials, wallet keys, automated order placement, or custody belongs
outside the current security model and requires a separate design review.
