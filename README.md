# pmwhale

Инструментарий для исследования стратегии **копирования «китов» на Polymarket**:
собрать публичные данные → отранжировать кошельки по риск-скорректированной
доходности → **честно** прогнать бэктест копирования с учётом задержки реакции
и проскальзывания.

> Работает только с **публичными** данными Polymarket (Gamma / Data / CLOB API),
> без авторизации и без приватного доступа. Это research-инструмент проверки
> гипотезы, а не готовая доходная схема. Ставки деньгами — под твою юрисдикцию
> и ответственность; это не финансовый совет.

---

## Идея

Наивное «кит зашёл крупно → повторяю» почти всегда убыточно после задержки и
комиссий. Ценность появляется, только если:

1. копировать не по **размеру** ставки, а по **исторической риск-скорректированной
   доходности** кошелька;
2. честно моделировать, что ты входишь **позже** кита и по **худшей** цене.

`pmwhale` даёт минимальный конвейер, чтобы это проверить на реальных данных.

## Структура

| Файл | Назначение |
|------|-----------|
| `client.py`   | Клиент к публичным API Polymarket (Gamma / Data / CLOB) + rate-limit и backoff |
| `collect.py`  | Сидирование китов (топ-холдеры крупнейших рынков) → сбор сделок в SQLite |
| `rank.py`     | Рейтинг кошельков: PnL, винрейт, sharpe-подобная метрика |
| `backtest.py` | Главный тест: PnL копирования при разной задержке входа |
| `docs/ENDPOINTS.md` | Проверенные вживую эндпоинты и форматы полей (дока Polymarket местами расходится с реальностью) |

## Установка

```bash
python -m venv .venv
. .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

## Использование

```bash
# 1. собрать данные: топ-рынки -> киты-холдеры -> их сделки -> pmwhale.db
python collect.py

# 2. рейтинг кошельков по риск-скорр. доходности
python rank.py

# 3. бэктест копирования (передай топ-кошельки из rank.py)
python backtest.py 0xКОШЕЛЁК1 0xКОШЕЛЁК2 ...
```

Параметры сбора настраиваются в `collect.main(n_markets=, holders_per_market=, min_amount=)`.

## ⚠️ Что доделать, иначе цифры бэктеста врут

Каркас рабочий, но для валидных цифр нужно закрыть известные упрощения
(помечены в коде как заглушки):

1. **Реальный сеттлмент.** `backtest.resolved_price()` — заглушка (берёт последнюю
   цену = look-ahead bias). Заменить на фактический исход рынка из
   `/closed-positions` или on-chain resolution.
2. **P&L из позиций, а не из сделок.** `/positions` и `/closed-positions` отдают
   готовые `realizedPnl / cashPnl / percentPnl` — точнее, чем реконструкция в
   текущем `rank.py`.
3. **Реальный слиппедж.** Сейчас константа; брать из глубины ордербука CLOB на
   момент входа. На тонких маркетах слиппедж съедает весь эйдж.
4. **Отбор без look-ahead.** Рейтинг кошельков считать только на данных **до**
   периода бэктеста.

Подробности и статус — в [docs/ENDPOINTS.md](docs/ENDPOINTS.md).
